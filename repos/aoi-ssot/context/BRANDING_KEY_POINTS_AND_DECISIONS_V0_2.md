## 1) Launch philosophy (Infra-first)
- Soft Launch를 우선한다. 단, 돈/지갑/무인 자동실행은 베타로 풀지 않는다.
- 안전 3대 원칙
  - Default = Watch-only / DRY_RUN / Simulate
  - LIVE = 명시적 사용자 확인(Confirm) + 하드캡 + allowlist + fail-closed
  - “수익/알파” 주장 금지 → proof & safety 포지셔닝

## 2) What we are
- ‘완성 앱’이 아니라 **운영체제/프로토콜/인프라**.
- PMF는 기능 수가 아니라 **표준(interfaces/schemas) + 증빙(proof archive) + 운영(risk gates/governance)** 3축이 자리 잡을 때.

## 3) Roadmap (launch stages)
- Stage A (Core): Run Artifact Standard
  - run report / manifest / proof / risk gate I/O 고정
- Stage B (Bazaar): 거래가 아니라 **플러그인 레지스트리**부터
  - DRY_RUN 기본, LIVE는 승인/확인 기반
- Stage C (DENX): 자동트레이딩이 아니라 **Decision Exchange**(결정+리스크+증빙)

## 4) Success definition
- 성공 = 단기 돈보다 **AI 인프라 선두주자(표준/신뢰/생태계) 선점**.
- 운영 루틴(지속가능)
  - 주 2회 Proof Day
  - 주 1회 Standard Day
  - 월 1회 Deprecation Day
