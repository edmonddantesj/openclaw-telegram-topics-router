# BUIDL CTC Hackathon — Submission Ideas (v0.1)

Hackathon: https://dorahacks.io/hackathon/buidl-ctc/detail
Track target (proposed): **DeFi**

## Idea 1 — "ProofPay: Pay-to-Deliver Commerce for Agents (Creditcoin Testnet)"
- One-liner: A proof-first agent commerce service where delivery is gated by payment verification, emitting a verifiable bundle (manifest+sha256) per order.
- Why Creditcoin: uses Creditcoin as a scalable settlement / execution environment for agentic commerce.
- Demo: create order → invoice/challenge → verify → deliver → show proof bundle.

## Idea 2 — "Ops-Receipts: Evidence Bundles for DeFi Automation"
- One-liner: A toolkit that wraps DeFi automations with receipts (artifacts + sha256), enabling auditability for every automated action.
- Why Creditcoin: positions Creditcoin as the chain where ops evidence + actions are recorded and replayed reliably.

## Idea 3 — "Shadow-to-Live DeFi Agent (Risk-Gated on Creditcoin)"
- One-liner: A DeFi agent that runs Shadow→Canary→Live with strict caps and governance gates, producing proofs at each stage.
- Why Creditcoin: highlights reliability + real-world deployment story on Creditcoin testnet.

## Minimum submission checklist (from page)
- GitHub repo (README required)
- Deck or whitepaper (PDF URL)
- Prototype demo video URL
- Testnet deployment proof
- Team member info
