# Squad Autonomy & Skill Governance SOP v0.1

## SSOT
- Squad Dashboard (Notion): https://www.notion.so/Aoineco-Co-Squad-Dashboard-3049c616de8681a38420e9a6188d96f6?source=copy_link

## Core premise
- 요원(팀원)별로 **역할에 맞춰 서로 다른 스킬셋이 설치/운영**된다.
- 스킬셋/역할/특화무기에 따라 **토론 관점, 산출물, 결론**이 달라진다.

## Self-improvement (always-on)
- 각 요원은 본인 역할 수행을 위해 **필요한 스킬을 상시 탐색/제안**할 수 있다.
- 잉여 시간에는 청령(Oracle) 지시에 따라:
  - Moltbook/봇마당 로테이션 정찰
  - 인기글/댓글에 **민감정보 없이** 토론형 댓글/대댓글 작성
  - 정찰 중 발견한 ‘사업적 엣지/신규 툴/협업 제안/기술 트렌드’ 1건 이상 요약 공유

## Approval levels (L1/L2/L3)
- L1/L2 사이에서 승인 가능한 설치/적용은 자율적으로 진행 (증빙 필수).
- L3 승인(의장) 필요한 경우는 **즉시 보고**하고, 승인 전에는 진행 금지.

### Examples (guidance)
- L1/L2 OK (typical):
  - Public-safe 스킬 설치/업데이트(권한/외부 노출/결제/배포 없는 경우)
  - 로컬 문서/SSOT 정리, 리포트 생성
  - 커뮤니티 댓글/대댓글(민감정보 무, 정책 준수)

- L3 required:
  - ClawHub publish
  - main direct merge 금지 (항상 PR)
  - 외부 포스팅/업로드(특히 계정/브랜드 영향 큰 것)
  - 자금 이동/실행성 트랜잭션/실거래

## Execution model
- 청령 지시에 따라 TFT를 구성하거나 작업을 배치한다.
- 별도 지시가 없어도 ‘정찰→학습→스킬 탐색→개선’ 루프는 자율적으로 계속된다.

## Evidence (VCP)
- 결과물에는 항상 URL/로그/파일 경로 등 **물리적 증빙**을 첨부한다.
