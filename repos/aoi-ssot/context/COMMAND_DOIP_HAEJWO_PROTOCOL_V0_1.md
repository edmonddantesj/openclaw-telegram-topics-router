# Command Protocol — “도입해줘” (V0.1)

> 업데이트: 산출물(Proof) 포맷까지 고정한 V0.2가 생성됨 → `context/COMMAND_DOIP_HAEJWO_PROTOCOL_V0_2.md`

## 0) 정의 (SSOT)
사용자가 스킬/레퍼런스 도입 논의 중 **“도입해줘”**라고 말하면, 에이전트는 아래를 **기본 의미(디폴트)**로 이해하고 실행한다.

## 1) 실행 순서 (반드시 이 순서)
1. **라이선스/보안 검증 (필수)**
   - 라이선스/상업적 제한 문구 **캡처/인용** (없으면 `미명기`로 기록)
   - 보안 스캔(권한/파일 접근/네트워크 송신/eval·exec/키 취급/자동 실행(hooks·cron)) + 결과 기록
   - 의심 패턴(토큰 탐색, 임의 업로드, 난독화 등) 발견 시: **L2로 승격** + 사용 보류(리빌딩으로 가드레일 강화)
2. **창작자 연락처 탐색/기록**
   - 공개된 이메일/연락수단이 있으면 기록
   - 없으면 **공란(TBD)** 으로 두되, “탐색 시도함” 상태를 로그로 남김
3. **리빌딩/디벨롭(우리 경험 반영) + S‑DNA 재구성 (보안 업그레이드 포함)**
   - 라이선스/보안 명기 여부와 무관하게, ‘우리 방식’으로 리빌딩/디벨롭 제작 진행
   - **보안 업그레이드 기본 포함:** allowlist/timeout/retry 제한, 민감정보 로그 마스킹·출력 금지, `.env` 차단, 위험 실행은 approve gate
   - 단, **상업적 활용 금지**가 명시된 경우: 내부 사용 범위에서만 리빌딩/활용(외부 공개/판매/배포 금지)
   - 결과물에는 **보안 변경내역(근거→패치 요약)**을 남겨 S‑DNA로 박제
4. **설치/내부 적용**
   - 기본은 **원본 그대로 설치가 아니라, 리빌딩 결과물을 설치**하여 활용

## 2) 외부 공개 금지 (Nexus Bazaar)
- **Nexus Bazaar 베타 오픈 전까지**: 리빌딩한 스킬(및 파생물)은 **외부 공개/배포/판매/게시 금지**

## 3) 승인 라우팅 (L1/L2/L3)
- **L1:** 내부 리스크 낮음 → 위 순서대로 진행 후 내부 적용
- **L2:** OAuth/외부API/권한 넓음/자기변형/보안 플래그 등 → 참모진 승인 게이트 후 진행
- **L3:** 재무/브랜드/법무/되돌리기 어려운 리스크 → 에드몽 승인

## 4) 기록 의무 (Notion + 로열티 준비)
- 모든 정찰/도입은 **Notion에 출처 URL 포함**하여 기록(분석 리포트 템플릿 준수)
- 라이선스 확인 시 **창작자 이메일/연락수단 기록**(+ 없으면 TBD)
- **정식 런칭/베타 오픈 2~3주 전**: 로열티/문의용 메시지 패키지 문서 완성 + **연락 시도 로그**(날짜/채널/결과/증빙) 최소 1회 이상 남김
  - 템플릿: `context/ROYALTY_OUTREACH_CONTACT_LOG_TEMPLATE_V0_1.md`
- 로열티/어트리뷰션 정책: `context/ROYALTY_AND_ATTRIBUTION_POLICY.md`
