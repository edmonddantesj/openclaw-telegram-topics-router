# AOI Squad Pro — First Customers & Pricing (v0.1)

## Positioning (1-liner)
**AOI Squad Pro is a proof-first Agent Ops OS: approve → run → proof.**

We sell *verifiable operations* (evidence + governance + risk controls), not “automation scripts.”

---

## First customer hypotheses (Top 2)
### H1) Onchain/Trading/Prediction operators (risk + audit)
**Who:** individuals/teams running bots/agents that touch funds or time-sensitive markets.
**Pain:** failures are expensive; hard to reproduce; risk controls missing; no receipts for decisions.
**Why buy:** Shadow→Canary→Live gates + Proof Bundle + daily digest = safer iteration.
**Entry wedge:** Limitless Shadow reference + canary caps template.

### H2) Agent teams / internal ops squads (multi-run governance)
**Who:** teams operating multiple agents/cron pipelines across Notion/Supabase/workflows.
**Pain:** ops sprawl, no standard logs, no audit trail, unclear approvals, brittle cron.
**Why buy:** governance tiers + standard evidence + SSOT sync reduces coordination cost.
**Entry wedge:** install + 10-minute quickstart + proof sample.

---

## MVP package (minimum sellable)
1) **10-minute Quickstart** (install → run → proof → digest)
2) **Masked Proof Bundle sample** (public-safe)
3) **Governance & risk**: L1/L2/L3 + Shadow→Canary→Live + caps + kill-switch template
4) **SSOT**: local + Notion OR Supabase (at least one)

## Objections & answers (battlecard)
- Objection: “We can just use cron/Zapier/agent framework.”
  - Answer: “Those run tasks. We ship **receipts + governance + risk gates**. When money/security/external is involved, that’s the product.”

- Objection: “We already log things.”
  - Answer: “Logs aren’t evidence. We standardize artifacts + sha256 + SSOT so ops is reproducible and auditable.”
