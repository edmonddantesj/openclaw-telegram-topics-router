## 0) Purpose
Lock public-facing naming so AOI Core / Nexus Protocol / Nexus Bazaar do not drift across documents, demos, and posts.

## 1) Canonical brand architecture (HARD)
- **Top-level product/platform:** **AOI Core**
  - One-liner: Policy → Gate → Approval → Proof
- **Reference architecture frame:** **Nexus Protocol (reference)**
  - Use only as an explanatory mapping (Identity/Intelligence/Economy)
  - Never as the top-level product name
- **Market layer (roadmap / TEASER):** **Nexus Bazaar**
  - Proof-first market, default report-only demos

## 2) Forbidden / constrained terms
- **Forbidden external top-level:** “AI DEX”
- Avoid “token swap DEX” language unless INTERNAL/TOP SECRET and explicitly approved.

## 3) Approved external phrasing (safe defaults)
- “AOI Core — auditable agent operations (Policy → Gate → Approval → Proof)”
- “Nexus Protocol (reference) to explain Identity/Intelligence/Economy mapping”
- “Nexus Bazaar (roadmap) — proof-first market; report-only by default”
