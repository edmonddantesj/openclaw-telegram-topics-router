# Skill Scouting Governance (L1/L2/L3) — Aoineco & Co.

## 목적
새로 정찰한 스킬/레퍼런스를 **빠르게 도입**하되, 위험·라이선스·운영 리스크는 단계적으로 통제한다.

## 3-Tier 승인 구조
- **L1 (자율):** 팀 내부 판단으로 즉시 도입 가능
- **L2 (참모진):** 이슈는 있으나 매력적인 아이템 → 참모진 기준으로 도입
- **L3 (의장/에드몽):** 재무/브랜드/법무/대외노출 리스크가 크거나, 되돌리기 어려운 변경 → 의장 승인 필수

## 운영 흐름 (기본)
1) **정찰 감지**: 새 스킬 후보 발견
2) **청령 배정**: 역할에 맞는 오너/백업 지정
3) **분석**: 오너가 (기능/의존성/데이터 흐름/권한/라이선스) 체크 + PoC(가능하면 로컬)
4) **등급 판정**: L1/L2/L3 중 하나로 분류
5) **도입(반드시 순서 준수)**:
   1) 라이선스/보안 검증(문구 캡처/인용; 미명기면 ‘미명기’로 기록)
   2) 창작자 연락수단 탐색/기록(있으면 기록, 없으면 TBD)
   3) 우리 경험 반영 리빌딩/디벨롭 + S‑DNA 재구성
      - (상업적 재제작/활용 금지 명시) → 내부용으로만 사용(외부 배포/판매/공개 금지)
   4) 설치/내부 적용(기본은 ‘리빌딩 후 설치’)
   5) **Nexus Bazaar 베타 오픈 전까지 외부 공개/배포 금지**
   - L1 → 내부 적용까지 자율
   - L2 → 참모진 승인 후 진행
   - L3 → 에드몽 승인
6) **기록(필수)**: Notion Reference Inbox에 출처 + 분석 리포트 양식으로 박제
   - 추가 의무: **창작자 연락수단(이메일/핸들/웹폼) 기록 + 정식 런칭 전 ‘연락 시도 로그’ 남기기**(날짜/채널/결과/증빙)

## L1 / L2 / L3 판정 가이드
### L1 (즉시 도입)
- 로컬/샌드박스에서만 사용, 권한 범위가 좁음
- 외부 송신/결제/삭제/계정권한 변경 없음
- 라이선스 상업화 금지 문구 없음(또는 내부용 사용에 문제 없음)

### L2 (승인 후 도입)
- 외부 API/네트워크 호출이 핵심이거나, 권한/데이터 범위가 넓음
- 운영 리스크(비용 폭주/Rate limit/키 관리)가 있음
- 그래도 제품 가치가 커서 빠른 실험이 필요

### L3 (의장 승인)
- 상업화/재배포 라이선스가 불명확하거나 제한이 명시됨
- 결제/지갑/토큰/자금 이동 또는 대외 브랜드 리스크
- 시스템 전역 설정 변경/보안 영역 접근/되돌리기 어려운 변경

## GitHub Push / S-DNA 업데이트 추가 규칙(필수)
- S‑DNA가 새겨진 스킬/레포는 불편함이 있으면 **꾸준히 업데이트**한다.
- 단, 원격 GitHub push는 반드시 **L2/L3 승인 게이트**를 통과해야 한다.
- 보안 감사 기준을 훼손하는 업데이트는 **절대 불허**.
- 허위정보/잘못된 코드 삽입은 **누적 경고(Strike)**로 관리하고, 도움이 없다고 판단되면 **단종/변경**한다.
- 상세 규정: `context/SDNA_UPDATE_AND_PUSH_GOVERNANCE.md`
