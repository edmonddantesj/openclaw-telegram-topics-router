# AOI SkillKit Naming Rule v0.1

## Goal
Keep SkillKit product line consistent and searchable.

## Canonical pattern
`SkillKit-<CHAIN>-<DOMAIN>`

- `<CHAIN>`: BSC | BASE | SOL | ETH | ... (UPPERCASE)
- `<DOMAIN>`: short PascalCase capability name (no spaces)

## Examples
- `SkillKit-BSC-DeFiExecution` (BNB Good Vibes Only)
- `SkillKit-BASE-OnchainExecution` (Base-safe execution skeleton)
- `SkillKit-SOL-DexOps` (Solana DEX ops runner)

## Notes
- Keep names neutral (no hype words).
- Chain+domain should describe *what it does*, not *why it’s cool*.
