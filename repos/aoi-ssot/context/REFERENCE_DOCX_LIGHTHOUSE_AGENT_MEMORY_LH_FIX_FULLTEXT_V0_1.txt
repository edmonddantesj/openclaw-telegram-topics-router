The Uncomfortable Truth About Agent Memory
Steinberger built OpenClaw with radical transparency. Plain Markdown files as the source of truth.
MEMORY.md
for long-term knowledge. Daily logs in memory/YYYY-MM-DD.md. Local SQLite for search indexing. Everything stored in ~/.openclaw/ on your machine.
It's elegant. You can read your agent's memory with a text editor. But here's what's also true.
Context compaction destroys memories. When sessions get long, OpenClaw compresses older context to save tokens. Learned preferences, project details, decision history: silently summarized, truncated, or dropped.
Sessions start from zero. Every restart re-reads whatever survived on disk. Miss a file and it's gone.
One disk failure means total memory loss. No replication. No backup. No redundancy.
Plain-text secrets sit at predictable paths. CrowdStrike, Cisco, and Palo Alto Networks all published security analyses flagging ~/.openclaw/ as a prime target. Bitsight found 30,000+ exposed instances on the open internet. Security researchers found 42,000+ unprotected gateways. Cisco found 26% of ClawHub skills contained vulnerabilities. VirusTotal scanning flagged 341 malicious skills, a 12% contamination rate.
The pain is so acute that four companies shipped memory plugins within weeks to patch it: Mem0, Cognee, Supermemory, Maximem. But every one of them routes your agent's brain through centralized servers.
And that leads to the next problem.

What Agents Actually Need: IPFS, Filecoin, and the Decentralized Stack
@IPFS
, the InterPlanetary File System, is peer-to-peer content-addressed storage. Files are identified by what they contain, not where they're hosted. Each file gets a CID, a Content Identifier, which is a cryptographic hash of the content. Change one byte and you get a different CID. Tamper-proof by design. Any node on the network can serve the file. When Wikipedia got blocked in Turkey, they put it on IPFS. Lockheed Martin is launching IPFS nodes into orbit.
@Filecoin
is the world's largest decentralized storage network. It uses economic incentives to maintain a global network of independent storage providers who keep your data alive, with cryptographic proofs they're actually doing it. No central company. No "US-EAST-1." When AWS went down for 15 hours, Filecoin kept running because there's nothing centralized to fail.
Together they form a complete storage layer. IPFS gives you integrity and distribution. Filecoin gives you persistence and redundancy. No single point of failure. No recurring fees. No vendor that can go offline and take your agent's brain with it.
The missing piece was always making this agent-native. Two protocols solve that.
MCP, the Model Context Protocol, is the open standard from Anthropic for connecting AI agents to external tools. It's how OpenClaw talks to everything: email, calendars, browsers, storage. Any storage that wants to be agent-native needs an MCP server.
x402 is the internet-native payment protocol from Coinbase. It lets agents make autonomous payments over HTTP using stablecoins. No accounts. No subscriptions. No human in the loop. Google's AP2 protocol incorporates it. Cloudflare and Vercel support it. Over $10M in volume on Solana. OpenClaw agents are already buying Linux VMs with USDC on Base via x402. (
@x402intern
)
Now imagine an agent that autonomously stores encrypted memories on a global decentralized network, retrieves them by content hash from anywhere, verifies they haven't been tampered with, and pays for the storage itself. All through standard protocols.
That's not theoretical. The stack is live.

What This Looks Like in Practice
