# Royalty & Attribution Policy — Aoineco & Co.

## 목적
- 스킬 도입/리빌딩/상품화 과정에서 **원작자(외부) 및 팀원(내부 창작자)**의 가치를 유지한다.
- 추후 **로열티 지급이 가능**하도록, 처음부터 출처·라이선스·기여·매출 연결 정보를 구조화해 기록한다.

## 적용 범위
1) 외부 스킬/레퍼런스(ClawHub/GitHub 등) 도입
2) 외부 스킬을 기반으로 한 리빌딩/리마스터(허용 범위 내)
3) 팀원들이 신규로 창작한 스킬을 판매하는 경우

## 필수 기록 항목(스킬 단위)
- Source URL (ClawHub/GitHub/Docs)
- Original author/owner (가능하면)
- **Contact methods (공개된 연락처/핸들/이메일)**
- **Contact attempt log (정식 런칭 전 ‘연락 시도’ 증빙: 날짜/채널/결과/스크린샷 또는 링크)**
- License / commercial restrictions (명시 문구 캡처/인용)
- Aoineco 변경 내역(요약) + S-DNA ID
- Maintainer(오너/백업) + 리뷰어
- Revenue linkage (상품/가격/결제 주소/정산 방식)
- Royalty plan (기본 % 또는 fixed, 지급 조건, 지급 주기)

## 로열티 원칙(초안)
- **외부 원작 기반 리빌딩:** 라이선스가 허용하는 범위에서만 진행. 허용 시에도 attribution 유지 + 로열티 옵션을 남긴다.
- **내부 신규 창작 스킬:** 팀원(요원)에게 판매 매출 기반 인센티브/로열티를 설계할 수 있다.

## 운영 방식
- Notion Reference Inbox: 모든 외부 스킬은 "🔍 상세 분석 리포트"에 **Attribution/Royalty 섹션**을 추가해 박제.
- **연락수단 + 연락 시도 로그**: 창작자 이메일/핸들 기록 + 정식 런칭 전 최소 1회 이상 ‘연락 시도’ 증빙을 남긴다.
- **사전 문서 준비(필수)**: 베타 오픈 **2~3주 전**까지 로열티/문의용 메시지 패키지(일괄 전송용) 문서를 완성한다.
  - 템플릿: `context/ROYALTY_OUTREACH_CONTACT_LOG_TEMPLATE_V0_1.md`
- 판매용 스킬(상품)로 승격 시: 별도 판매/정산 DB(또는 페이지)에 royalty plan을 연결.

## 주의
- 라이선스에서 **재배포/상업화 금지**면: 내부 사용만. 로열티로 해결하려 하지 말고, 허용 라이선스/재작성으로 전환.
- 로열티 지급은 장부/증빙 기반으로(임의로 약속하지 말고 문서화).
