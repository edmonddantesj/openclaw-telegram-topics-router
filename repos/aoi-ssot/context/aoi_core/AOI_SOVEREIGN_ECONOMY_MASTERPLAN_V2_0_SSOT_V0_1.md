## 2. Regulatory Hardening Framework

### 2.1 Core Principle
**"$AOI는 '증권(Security)'이 아닌 '소비형 유틸리티 토큰(Consumptive Utility Token)'이다."**

### 2.2 Global Compliance Matrix
| 규제 체계 | 관할 | 핵심 대응 전략 |
| :--- | :--- | :--- |
| **MiCA (EU)** | 유럽연합 | Title II Utility Token으로 분류. 발행 전 NCA 통지 및 14일 철회권 구현. |
| **디지털자산법 (KR)** | 대한민국 | 가상자산으로 분류하되, '공동사업 참여' 및 '수익 배분' 요소를 배제하여 증권성 회피. |
| **Howey Test (US)** | 미국 | "타인의 노력에 의한 수익 기대" 요건 불충족. 서비스 크레딧 선구매(SCFP) 모델 도입. |

### 2.3 Compliance Checklist (Pre-Token Launch)
- [ ] **기능 제공 의무:** 토큰 발행 시점에 Brand-Genesis/Smart-Manager 등 실질적 서비스가 라이브 상태여야 함 (MiCA Art. 4(3)).
- [ ] **마케팅 통제:** "투자", "수익", "가격 상승" 단어 사용 전면 금지 (COMMS_V0_1 준수).
- [ ] **환불 메커니즘:** 14일 철회권을 스마트 컨트랙트 레벨에서 구현.
- [ ] **투명성:** 팀 물량 및 락업 일정을 온체인에 공개하여 불공정거래 의심 차단.
