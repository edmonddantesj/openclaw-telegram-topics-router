# AOI Squad Pro 자동실행 프레임워크 (v0.1)

**요청 반영:** 청묘/청령이 직접 DM 발송 및 태스크 수행 오케스트레이션, 별도 DM 확인/회신 루프 대신 개별 담당자가 자체 수행 후 결과 제출.

## 1) 운영 원칙
- 본 오퍼레이션은 **L1/L2 범위**로 제한.
- 별도 L3 승인 없이 운영 가능. 단, 아래 금지 조건 발생 시 즉시 차단 후 중단 후 보고:
  - 외부 자금 이동/결제/송금/지갑 서명
  - 외부 공개 유출 가능성(비밀 데이터, 키, 토큰, 개인 식별정보)
  - 되돌리기 어려운 destructive file operation(`rm -rf`, 권한 변경, 브랜치 강제 push 등)
- 모든 결과는 “증빙 가능한 템플릿”으로 기록.

## 2) 진행 책임
- **청묘(오퍼레이터):** 전체 오케스트레이션 톤 세팅 + 최초 출발 안내 공지 + 최종 집계
- **청령(Oracle):** 1차 수신/분산, 진행 상태 체크, 미완료 항목 재촉
- **각 역할(청음/청검/청안/청비/청섬/청성/청기/청정 등):** 개별 템플릿 보고서 제출

## 3) 기본 루틴(반복)
1. **발송(청묘/청령 역할):** 이번 라운드 태스크 확정 메시지 1회만 발송
2. **실행(개별 멤버):** 10분 이내 1차 결과 제출(필요시 더 진행)
3. **수집:** 모든 멤버는 동일 템플릿에 필수 항목 채워 응답
4. **집계(청령):** 라운드 리포트 파일에 그대로 붙여넣기
5. **확인:** 미입력은 5분 내 재요청, 2회 미응답 시 라운드 제외 처리
6. **보관:** `memory/2026-02-19.md` + `CURRENT_STATE.md` + 라운드 리포트에 증빙 링크 남김

## 4) 경량 결과 템플릿(강제 필드)
각 멤버 반드시 아래 항목을 채움
- `Task ID`
- `Task Name`
- `Status`: `DONE / BLOCK / NEEDS_SUPPORT`
- `핵심 산출물`
- `수행 근거(1줄)`
- `증빙`
  - `proof_dir`(권장: `/tmp/aoi_squad_pro_run_<ts>/`)
  - `input_digest`
  - `sha256`
  - `logs`(경로)
- `다음 액션`
- `예상 리스크 / 제안`

## 5) DM 템플릿 (개별 발송용)
아래를 청묘가 한 번에 복붙해 각 멤버에게 발송

```text
[AOI Squad Pro — 자동 수행 요청]

이번은 너만 보면 된다. 별도 DM 반복 없이, 아래 템플릿으로 10분 안 1차 리포트 제출.

Task ID: 
Task Name: 
우선순위: P0/P1/P2
제한: 오직 회귀 가능한 local 증빙 작업만

필수 결과:
- Task 요약(1~2줄)
- 작업 상태(DONE/BLOCK/NEEDS_SUPPORT)
- proof_dir (권장: /tmp/aoi_squad_pro_run_<ts>/)
- input_digest
- sha256
- logs path
- 실패 시 failure_code, suggested_fix
- 다음 액션 1개

규칙:
- 승인 요청이 필요한 변경은 diff 먼저 제안, 승인 후 적용
- 미완료면 "BLOCK" + 사유 + 지원 필요 항목만 남기고 즉시 보고

제출 마감: +10분
```

## 6) 청렴 운영(중복/누락 방지) 규칙
- 같은 항목 중복 회신 불가: 1건이 최종안(필요하면 `UPDATE` 1회만 허용)
- 증빙 없는 DONE는 `BLOCK` 처리
- `input.json/proof.json/proof.sha256` 존재 유무가 불충분하면 `needsupport`
- `failure_code`가 있으면 `STATUS`를 DONE에 올리지 않음

## 8) 에스컬레이션(중요)
- 미응답(2회): 청령이 1회 차 재요청, 2회 차 미응답 시 라운드 제외 및 다음 할당으로 교체
- L3 사안(돈/보안/외부노출/파괴적 동작) 발견 시 즉시 중단 + 현 상태/증거 저장
- 20분 경과 후에도 P0가 1개 이상 미완료면 긴급 점검 모드

## 9) 추천 실행 순서 (오늘 바로)
- 1) 청묘 공지문 고정 발송
- 2) 역할별 태스크 1차 배치(3~4개씩)
- 3) 10분 후 자동 집계 탭 채우기
- 4) BLOCK/NES_SUPPORT 항목만 청령 보류 풀로 넘김
- 5) 완료분만 CURRENT_STATE + memory에 반영
