# Brand-Genesis — Product SSOT v0.1

Last updated: 2026-02-20 (KST)
Status: DRAFT
Exposure: OPEN (scope/outputs) / TOP SECRET (token pricing if any)

> Purpose: Define Brand-Genesis as an AOI Core-powered deliverable with proof-first governance.

---

## 1) What it is (definition)
**Brand-Genesis** is a structured engagement that produces a brand operating system (positioning + governance + proof-first workflow) for a project/team.

It is *not* marketing fluff: outputs must be auditable artifacts with evidence paths.

---

## 2) Inputs (required)
- Project context: 1–5 pages (or links) describing product + constraints
- Existing brand assets (if any): logo, name candidates, copy, domain, socials
- Governance constraints: what is L3 (money/signing/external publish)
- Exposure tier posture: OPEN/TEASER/STEALTH/TOP SECRET

---

## 3) Outputs (artifact list)
### Core pack (always)
1) `brand_one_pager.md`
   - One-liner, positioning, ICP, promise boundaries, forbidden claims
2) `naming_rules.md`
   - naming system + term mapping + do/don’t copy blocks
3) `exposure_tiers.md`
   - OPEN/TEASER/STEALTH/TOP SECRET mapping for the project
4) `governance_ops.md`
   - Policy→Gate→Approval→Proof 적용 방식
5) `proof_manifest.json`
   - generated_at, inputs_digest, evidence_paths, sha256 list

### Optional modules
- `pricing_outline.md` (TOP SECRET if token-linked)
- `website_copy_v0.md`
- `pitch_deck_outline.md`

---

## 4) Governance / safety
- Default: report-only + drafts.
- External posting requires explicit approval (L3).
- Never promise profit/alpha.

---

## 5) Success criteria (acceptance)
- All outputs exist and are internally consistent.
- Exposure tiers respected.
- Proof manifest reproducible (hashes match).
