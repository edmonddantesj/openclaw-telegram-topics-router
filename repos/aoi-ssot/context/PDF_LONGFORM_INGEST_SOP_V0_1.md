# PDF / Longform Ingest SOP v0.1 (Aoineco & Co.)

## 0) 목적 / 원칙
- **SSOT(원문 진실원본)**: Notion `📥 Reference Inbox (PDF/Longform)`
- GitHub에는 **원문 전체**보다 **가공본(요약/ADR/결정/증빙 링크)** 위주로 남긴다.
- 외부 게시(봇마당/Moltbook 등)는 **반드시 승인 게이트**를 통과한다: 마지막에 `올릴까? (Yes/No)`

---

## 1) 입력 형태 (무엇이 와도 OK)
- GitHub URL / repo
- PDF/문서 파일
- 긴 글 링크
- 텍스트 붙여넣기

---

## 2) 처리 순서 (고정)
### Step 0 — 1차 스캔(자동 중요도 분류)
- 사용자가 별도 멘트를 안 붙여도, **내가 문서를 1차로 훑고 중요도(S/A/B/C)를 자동 판정**한다.
- 판정 기준(요약):
  - **S:** 돈/보안/법/대외발표/비가역 의사결정 직결
  - **A:** 제품/운영/SOP/핵심 전략에 직접 도움
  - **B:** 벤치마킹/학습/아이디어 (재사용 가능)
  - **C:** 가벼운 참고/스캔 (저장 최소)

### Step A — Notion에 먼저 박제 (원문 SSOT)
**DB:** `📥 Reference Inbox (PDF/Longform)`

**필수 필드(속성):**
- Name
- One-liner
- Source URL
- Source Date
- Author
- Type / Tags / Status / Confidence / Business Fit

**본문(Children) 구성:**
1) **핵심 요약 5~10줄**
2) **근거/인용** (필요 시: 핵심 구절 + 출처 위치)
3) **우리 결론**
4) **다음 액션** (적용 아이디어 1~3개)

**원문 보관(중요도 기반 정책):**
- **S/A:** 원문(fulltext/파일)은 **로컬만 보관**(Notion에는 첨부하지 않음). Notion에는 요약/결론/액션/출처만.
- **B:** 기본은 로컬 보관, Notion에는 요약만(선택).
- **C:** 로컬/Notion 모두 **원문 저장하지 않음**(필요 시 출처 링크만).

### Step B — 3종 세트 산출물 생성 (항상)
#### (1) 내부 분석글 — 벤치마킹 리포트 (KR)
포맷(필수 섹션):
- 한 줄 요약
- 핵심 구조(무엇이 먹히는지)
- 우리가 가져갈 요소(제품/운영/세일즈/콘텐츠)
- 리스크/주의사항(과장, 정책, 컴플라이언스 등)
- 다음 액션(우리 쪽 적용 아이디어 1~3개)

#### (2) Moltbook 초안 (EN)
- 짧고 임팩트, **중립 톤**, 글로벌 독자 기준
- 출처 포함(허용 범위 내 링크/핸들)

#### (3) 봇마당 초안 (KR)
- 짧고 임팩트, **중립 톤**
- 출처 포함

### Step C — 승인 게이트 (필수)
- 항상 마지막에 묻는다:
  - **"올릴까? (Moltbook Yes/No, 봇마당 Yes/No)"**
- 승인 전:
  - **업로드/포스팅 0**

> ✅ 변경(2026-02-19): 장문 PDF/DOCX 인입 시 **원문을 채팅에 그대로 재출력하지 않는다**.
> - 원문 SSOT는 Notion(첨부/URL) + 로컬 파일로만 보관
> - 채팅에는 요약/초안/핵심 인용(필요 시 일부)만 제공

---

## 3) GitHub에 남길 때 기준
**원칙:** GitHub에는 “장문 원문”보다 가공본(요약/ADR/스펙) 위주

- 추천 경로:
  - `docs/` : 심사/외부 공개 가능한 문서
  - `context/` : 내부 SSOT(운영/정책/템플릿)
- 장문 원문을 그대로 올려야 하면:
  - 텍스트: `docs/<topic>.md` 가능
  - PDF/대용량 바이너리: **Notion/Release/외부 스토리지**에 두고 GitHub에는 **링크 + 요약**만

> 주의: 워크스페이스는 과거 git 오염 이슈가 있었으므로 **큰 파일 커밋은 보수적으로**.

## 4) 민감정보 / 컴플라이언스 체크 (자동 게이트)
다음은 **Notion이라도 분리/비공개** 권장, **GitHub에는 절대 업로드 금지**:
- 회사 내부 전략/사업계획/가격/협상/고객정보
- 시크릿/토큰/API 키/패스워드
- 지갑 관련 민감정보(프라이빗키, 시드, 니모닉)
- 내부 운영상 비공개 정책 문서(런칭 전)

- 외부 게시가 필요하면 반드시 **승인** 받고 진행.
